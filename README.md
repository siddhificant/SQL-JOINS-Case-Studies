# SQL JOINS Case Studies

# About This Project

This repository documents my journey of learning SQL JOINs by solving real-world business scenarios.

Instead of only writing queries, I focused on:
- Understanding data relationships
- Interpreting query outputs
- Deriving business insights

The project emphasizes analytical thinking and demonstrates how SQL can be used not just for data retrieval, but for solving operational and business-related problems.

# Key Learnings

- JOIN direction is often more important than syntax.
- LEFT JOIN + NULL conditions help identify missing or unmatched records.
- Writing SQL queries is only one part of analysis — interpreting the output is equally important.
- Relational data becomes more meaningful when connected to business impact.
- SQL can be used to uncover operational inefficiencies and decision-making insights.
